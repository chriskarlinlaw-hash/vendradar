AI Recommendations Feature

Pro tier killer feature: "Find the 10 best gym locations in San Diego"

Technical Approach:

  1. User inputs: city/area + category (gym) + radius
  2. Backend generates grid of lat/lng points (every 0.1 miles)
  3. Score each point using existing algorithm (parallelized)
  4. Sort by score, return top 10-20 with reasoning
  5. OpenAI GPT-4 generates reasoning per location

Cost Estimation:

  • Grid scan of 10-mile radius = ~300 points to score
  • 300 Census API calls: free
  • 300 Places API calls: ~$10 (within free credits at low volume)
  • 1 OpenAI API call (batch reasoning): ~$0.05
  • Total per recommendation: ~$0.10-0.20

Margin Analysis: $79/mo Pro tier → ~10 recommendations/mo = $0.10 × 10 = $1-2 cost → 97% gross margin maintained

Database Schema (Supabase PostgreSQL)

-- Users table
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email VARCHAR(255) UNIQUE NOT NULL,
  subscription_tier VARCHAR(20) DEFAULT 'free',
  stripe_customer_id VARCHAR(100),
  searches_this_month INT DEFAULT 0,
  contact_reveals_remaining INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Searches table (analytics + rate limiting)
CREATE TABLE searches (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id),
  address TEXT,
  category VARCHAR(50),
  score INT,
  lat DECIMAL(10,8),
  lng DECIMAL(11,8),
  searched_at TIMESTAMP DEFAULT NOW()
);

-- Competitor machines (user-contributed)
CREATE TABLE competitor_machines (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  reported_by UUID REFERENCES users(id),
  lat DECIMAL(10,8),
  lng DECIMAL(11,8),
  category VARCHAR(50),
  notes TEXT,
  verified BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Subscriptions table
CREATE TABLE subscriptions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id),
  stripe_subscription_id VARCHAR(100) UNIQUE,
  status VARCHAR(20),
  tier VARCHAR(20),
  current_period_start TIMESTAMP,
  current_period_end TIMESTAMP,
  cancel_at_period_end BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW()
);

Scoring Algorithm (Proprietary)

Location score calculated from 4 components (0-100 total):

function calculateLocationScore(location, category, data) {
  // 1. Foot Traffic (30 points)
  const poiCount = countNearbyPOIs(location, 0.5); // within 0.5mi
  const transitProximity = getTransitDistance(location);
  const footTrafficScore = Math.min(30, 
    (poiCount / 20) * 20 + // POI density
    (1 / transitProximity) * 10 // Transit access
  );

  // 2. Demographics (25 points)
  const incomeScore = normalizeIncome(data.medianIncome, category);
  const populationDensity = data.population / data.areaKm2;
  const demographicsScore = Math.min(25,
    incomeScore * 0.6 + 
    (populationDensity / 10000) * 10
  );

  // 3. Competition (25 points)
  const nearestCompetitor = findNearestCompetitor(location, category);
  const competitionScore = Math.min(25,
    (nearestCompetitor.distanceMiles / 2) * 25
  );

  // 4. Building Type Fit (20 points)
  const buildingFit = calculateBuildingFit(location, category, data.pois);
  
  // Category-specific weight adjustments
  const weights = getCategoryWeights(category);
  
  return Math.round(
    footTrafficScore * weights.traffic +
    demographicsScore * weights.demographics +
    competitionScore * weights.competition +
    buildingFit * weights.building
  );
}

// Category-specific weight adjustments
function getCategoryWeights(category) {
  const weights = {
    'gym': { traffic: 1.1, demographics: 1.2, competition: 0.9, building: 0.8 },
    'office': { traffic: 1.0, demographics: 1.0, competition: 1.0, building: 1.3 },
    'hospital': { traffic: 1.3, demographics: 0.8, competition: 0.9, building: 1.1 },
    // ... etc for all 8 categories
  };
  return weights[category] || { traffic: 1, demographics: 1, competition: 1, building: 1 };
}

Risk Assessment & Mitigation

Technical Risks

Risk: Google Maps API costs spiral out of control

Likelihood: Medium | Impact: High

Mitigation:

  • Implement aggressive caching (24h for static data)
  • Set hard billing caps in Google Cloud Console
  • Rate limit free users more strictly
  • Backup plan: Migrate to Mapbox (cheaper at scale)

Risk: Free data sources provide inaccurate scores

Likelihood: High | Impact: Medium

Mitigation:

  • Clearly label as "estimates based on public data"
  • Gather user feedback on score accuracy
  • Prioritize Foursquare integration (Phase 2)
  • Build credibility with testimonials, not just data accuracy

Risk: Supabase free tier limits hit sooner than expected

Likelihood: Low | Impact: Low

Mitigation:

  • Monitor connection count daily
  • Implement connection pooling
  • Upgrade to Pro ($25/mo) if approaching limits
  • Cost is only 0.3% of revenue at 100 customers

Business Risks

Risk: Operators don't trust AI recommendations

Likelihood: Medium | Impact: High

Mitigation:

  • Show transparent reasoning for every score
  • Start with manual lookup (builds trust before AI)
  • Offer free validation: "Submit your current location, we'll score it"
  • Collect testimonials from beta users

Risk: Market too small or operators won't pay

Likelihood: Low | Impact: Critical

Mitigation:

  • Already validated demand via r/vending research (A- grade)
  • Free tier proves value before asking for payment
  • Clear ROI: one good location = 6-24 months of subscription
  • Backup plan: expand to adjacent markets (ATMs, micro-markets)

Risk: Competitor copies the idea

Likelihood: Medium (6-12 months) | Impact: Medium

Mitigation:

  • First-mover advantage in blue ocean market
  • Build network effects quickly (user-contributed competition data)
  • Domain expertise moat (vending-specific scoring hard to replicate)
  • Ship fast, iterate weekly, stay 6 months ahead

Google Places API (Nearby Search)

// Server-side API route (Next.js)
export async function POST(request) {
  const { lat, lng, category } = await request.json();
  
  // Map category to Google Places type
  const typeMap = {
    'gym': 'gym',
    'office': 'office',
    'hospital': 'hospital',
    'school': 'school'
  };
  
  const response = await fetch(
    `https://maps.googleapis.com/maps/api/place/nearbysearch/json?location=${lat},${lng}&radius=800&type=${typeMap[category]}&key=${process.env.GOOGLE_PLACES_API_KEY}`
  );
  
  const data = await response.json();
  
  return NextResponse.json({
    poiCount: data.results.length,
    pois: data.results.map(p => ({
      name: p.name,
      type: p.types[0],
      rating: p.rating
    }))
  });
}
